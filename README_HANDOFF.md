# README_HANDOFF

Abra este arquivo primeiro.

## Escolha seu caminho

### User
Para entender o estado atual e o que ainda faz sentido fazer.
- Ler: [`CONTINUATION_GUIDE.md`](CONTINUATION_GUIDE.md)
- Depois: seguir os links recomendados no guia

### Agent
Para trabalhar com contexto técnico e regras do projeto.
- Ler: [`AGENT_GUIDE.md`](AGENT_GUIDE.md)
- Depois: seguir o fluxo de decisão e executar

### Dev
Para navegar pelos guias e partir para implementação rápida.
- Ler: [`GUIDES_INDEX.md`](GUIDES_INDEX.md)
- Depois: usar [`QUICK_REFERENCE.md`](QUICK_REFERENCE.md) para comandos e validações

### Time-constrained
Para quem precisa decidir rápido e agir em poucos minutos.
- Ler: [`QUICK_REFERENCE.md`](QUICK_REFERENCE.md)
- Depois: executar a validação que couber no tempo disponível

## Execução padrão

1. Abrir o arquivo do seu caminho.
2. Seguir os links indicados.
3. Executar a ação escolhida.
4. Validar o resultado.
5. Commitar se houver mudança.

## Regra prática

- Se estiver em dúvida, comece por `Time-constrained`.
- Se for agente ou Codex, comece por `Agent`.
- Se quiser contexto de sessão, comece por `User`.
- Se quiser ir direto para implementação, use `Dev`.
